# 🧠 FlashCard Engine

> Turn any PDF into a smart, practice-ready deck of flashcards — powered by AI and spaced repetition.

Live Demo: https://flash-card-engine-mauve.vercel.app

---

## What is this?

FlashCard Engine is a full-stack flashcard app built for Cuemath's AI Builder Challenge (Problem 1). 

A student drops in a PDF — a chapter on quadratic equations, class notes on the French Revolution — and gets back a clean set of cards ready to practice. The app then uses the SM-2 spaced repetition algorithm to schedule reviews, showing difficult cards more often and easy cards less often.

---

## Features

- **PDF to Flashcards** — Upload any text-based PDF and get 15-20 high quality flashcards generated by AI in seconds
- **Smart Questions** — Cards cover key concepts, definitions, relationships, and examples — not shallow surface-level questions
- **SM-2 Spaced Repetition** — Rate each card as Again / Hard / Good / Easy and the algorithm schedules the next review automatically
- **Progress Dashboard** — See mastered cards, shaky cards, due today, and overall mastery percentage
- **User Authentication** — Every user has their own private decks, invisible to others (self-initiated feature — see below)
- **Drag and Drop** — Drop your PDF directly onto the upload zone
- **Dark UI** — Clean, modern dark theme

## Key Decisions & Tradeoffs

### AI Model — Groq + LLaMA 3.3-70b
Chose Groq over OpenAI/Claude because it's completely free with generous limits, extremely fast (tokens per second), and LLaMA 3.3-70b produces high quality educational content. The prompt uses a strict system message to ensure cards test understanding, not just surface memory.

### SM-2 Algorithm
Implemented the SM-2 spaced repetition algorithm — the same one used by Anki, proven by decades of cognitive science research. After each card, the user rates 0-5 and the algorithm adjusts the next review interval. Cards rated poorly come back in 1 day. Cards rated well come back in weeks.

### PDF Extraction — unpdf
Tried `pdf-parse` (failed with Next.js server components), then raw buffer conversion (read PDF metadata garbage instead of content), finally settled on `unpdf` which reliably extracts clean text from any text-based PDF.

### User Authentication (self-initiated)
The problem statement didn't require authentication. But without it, every user would see everyone else's decks — making the app unusable in a real scenario. I added Supabase Auth with instant signup (no email verification) so users get a private workspace immediately. This was my own initiative because a good product should protect user data by default, not as an afterthought.

## Interesting Challenges

**PDF extraction was the hardest part**
Three different approaches failed before finding one that worked. `pdf-parse` crashed with Next.js Turbopack. Raw buffer conversion read internal PDF syntax instead of content. `unpdf` finally solved it cleanly.

**Gemini API issues**
Started with Google Gemini but hit model availability errors (404s on multiple model names) due to API key project linking issues. Switched to Groq which worked immediately and is faster.

**Memory errors**
Next.js Turbopack hit memory limits during development. Solved with `NODE_OPTIONS=--max-old-space-size=4096`.

**Card quality**
Initial cards were shallow and sometimes asked about document metadata ("What is the title?"). Fixed by adding a strict system prompt that explicitly bans document-structure questions and forces exam-style questions only.

## Running Locally

```bash
# Clone the repo
git clone https://github.com/YOURUSERNAME/FlashCard-Engine.git
cd FlashCard-Engine

# Install dependencies
npm install

# Create .env.local with your keys
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SECRET_KEY=your_supabase_secret_key
GROQ_API_KEY=your_groq_api_key

# Run development server
npm run dev
```

Open http://localhost:3000

## Database Schema

```sql
create table decks (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users(id),
  name text not null,
  card_count integer default 0,
  created_at timestamp default now()
);

create table cards (
  id uuid default gen_random_uuid() primary key,
  deck_id uuid references decks(id) on delete cascade,
  question text not null,
  answer text not null,
  ease_factor float default 2.5,
  interval integer default 1,
  repetitions integer default 0,
  next_review timestamp default now(),
  created_at timestamp default now()
);
```
